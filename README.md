# 智能日历生成器 (Smart Calendar Generator)

## 项目概述

每个月都有一些固定要做的工作，比如每月25日进行盘点，但25日可能是周末或假期，需要提前一天完成，基于这个理由，设计了**智能日历生成器**。

**智能日历生成器**是一个纯前端Web应用，用于生成符合iCalendar标准（.ics）的日历文件。用户可以根据自定义规则，批量生成一年12个月的定期任务日历。

本项目代码由[DeepSeek](https://www.deepseek.com/)生成

### 核心功能

- **灵活的工作日规则**：支持自定义基准日期，自动计算每月该日期之前最近的工作日（若当天为工作日则取当天）
- **完全可定制**：可自由设置年份、任务时间、任务名称、描述和分类标签
- **一键导出**：生成标准.ics文件，兼容Outlook、Google日历、Apple日历等主流日历应用
- **实时预览**：参数调整后可立即查看全年12个月的实际执行日期
- **零依赖**：纯HTML/CSS/JavaScript实现，无需后端服务

### 适用场景

- 企业定期任务（如每月盘点）
- 定期设备巡检、维护任务
- 月度报表提交、会议安排
- 任何需要“每月固定日期附近的工作日执行”的周期性任务

## 使用指南

### 快速开始

1. 打开网页（直接双击HTML文件或在浏览器中打开）
2. 根据需求配置参数：
   - 选择年份
   - 设置基准日期（如25号）
   - 设定任务开始和结束时间
   - 填写任务名称、描述和分类标签
3. 点击“刷新预览”查看全年各月的实际执行日期
4. 确认无误后，点击“下载.ics日历文件”
5. 将下载的.ics文件导入您的日历应用

### 参数说明

| 参数     | 说明                                                       | 示例                 |
| -------- | ---------------------------------------------------------- | -------------------- |
| 年份     | 生成日历的年份                                             | 2026                 |
| 基准日期 | 每月几号作为参考日期，系统会自动查找该日期之前最近的工作日 | 25                   |
| 开始时间 | 每日任务的开始时刻                                         | 09:00                |
| 结束时间 | 每日任务的结束时刻                                         | 09:15                |
| 任务名称 | 日历事件的标题                                             | 定期盘点             |
| 任务描述 | 详细描述（可选）                                           | 每月盘点，持续15分钟 |
| 分类标签 | 用于日历分类，多个标签用英文逗号分隔                       | 盘点,维护            |

### 规则说明

**工作日查找逻辑**：

- 输入基准日（如每月25日）
- 系统检查该日期是否为工作日（周一至周五）
- 若是，则选取该日期
- 若不是，则向前查找最近的工作日（如25日周六则取24日周五，25日周日则取23日周五）
- 若基准日超出当月天数（如2月30日），则自动取当月最后一天作为基准

**示例**：

- 2026年1月25日（周日）→ 取1月23日（周五）
- 2026年2月25日（周三）→ 取2月25日（周三）
- 2026年3月25日（周三）→ 取3月25日（周三）

### 日历导入方法

#### Outlook

1. 打开Outlook日历
2. 点击“文件” → “打开并导出” → “导入/导出”
3. 选择“导入iCalendar (.ics) 或vCalendar文件”
4. 选择下载的.ics文件并导入

#### Google日历

1. 打开Google日历网页版
2. 点击右侧“设置”图标 → “设置”
3. 选择“导入并导出”
4. 选择.ics文件并导入

#### Apple日历

1. 打开日历应用
2. 点击“文件” → “导入”
3. 选择.ics文件
4. 选择目标日历并确认

## 技术栈

### 前端技术

- **HTML5**：页面结构
- **CSS3**：样式与响应式布局（Flexbox、Grid）
- **原生JavaScript (ES6+)**：业务逻辑、DOM操作、日历生成

### 核心算法

- **工作日判断**：基于JavaScript Date对象的getDay()方法，判断周一至周五
- **向前查找**：从基准日向前逐日遍历，找到第一个工作日
- **ICS生成**：严格遵循RFC 5545标准，生成完整的iCalendar格式文件

### 兼容性

- 支持所有现代浏览器（Chrome、Firefox、Safari、Edge）
- 生成的.ics文件兼容Outlook 2016+、Google日历、Apple日历、Thunderbird等

### 依赖

- 无外部依赖，纯原生实现

### 一、新增功能

#### 1. 仅生成剩余月份日历
**功能说明**：生成ICS文件时，只包含当前年份中从当前月份开始至年底的日历任务，已过月份不再导出。

**实现方式**：
- 新增 `getCurrentMonth()` 函数获取当前月份（1-12）
- 新增 `getRemainingInventoryDates()` 函数，根据所选年份与当前年份的对比：
  - 所选年份 < 当前年份：返回空数组（全部过期）
  - 所选年份 = 当前年份：从当前月份开始生成
  - 所选年份 > 当前年份：生成全年12个月
- 下载时使用 `currentRemainingDates` 而非 `currentAllDates`

#### 2. 预览区显示全年并标注过期月份
**功能说明**：预览时显示全年12个月的实际执行日期，已过月份进行灰色样式标注并显示“已过期”标签。

**实现方式**：
- 新增 `isMonthExpired(year, monthIndex)` 函数判断月份是否过期
- 新增 `getAllInventoryDates()` 函数生成全年12个月数据
- 预览渲染时：
  - 为过期月份添加 `expired` CSS类（灰色背景、降低透明度）
  - 添加“📅 已过期”徽章
  - 顶部信息横幅显示过期月份数量和可导出月份数量

#### 3. 预览按钮控制显示
**功能说明**：预览区域默认隐藏，点击“预览全年日历”按钮后才显示预览内容和下载按钮。

**实现方式**：
- 预览区域 `div` 初始 `display: none`
- 点击预览按钮时添加 `show` 类，切换为 `display: block`
- 下载前检查预览区域是否已显示，若未显示则提示用户先预览

#### 4. 信息横幅提示
**功能说明**：预览时显示当前年份、过期月份数量、可导出任务数量等信息。

**实现方式**：
- 新增 `infoBanner` 元素
- 在 `renderPreview()` 中动态计算 `remainingCount` 和 `expiredCount`
- 根据是否有过期月份显示不同提示文案

### 二、移除/修改内容

#### 1. 移除盘点专用描述
**原内容**：任务名称默认“盘点”，描述包含“盘点”相关文字
